Decision Brief

內湖淹水更像流域排水瓶頸

現有資料較不支持「內湖本地極端暴雨」或「基隆河主河道達警戒倒灌」作為單一主因；風險主軸是長時間降雨後，整個集水區逕流同時進入內水排水系統。

目前判斷

核心問題較可能是內水排除不及與流域排水瓶頸。

AI 分析測試資料

本網站為強降雨與淹水風險的初步分析展示，屬於 AI 協作產生的測試性資料，並非政府機關公告、工程鑑定報告或最終責任判定。

本頁內容由使用者提供雨量、水位、抽水站 API 等公開或截圖資料後，透過 OpenAI Codex 協助整理、分析與製作成決策儀表板。所有結論應作為討論與追查方向，仍需以主管機關原始資料、逐分鐘運轉紀錄與專業水利技師判讀為準。

初步技術結論

內湖淹水的核心問題，不是雨只在內湖下太大，也不是目前資料可證明的基隆河主河道外水氾濫。

較合理的初步判斷是：長時間降雨後，上游山區產流、本地降雨、排水幹線匯流、抽水站操作條件與基隆河外水位共同作用，使道路側溝、雨水下水道、幹線箱涵、抽水站進水端、閘門與抽水調度之間的某一個或多個環節形成瓶頸。

可支持內湖本地雨強不極端，上游山區雨量與持續時間才是主要產流訊號。
可排除部分可能南湖大橋、大直橋水位未達三級警戒，不支持主河道已達警戒作為單一主因。
仍需保留抽水站 API 是最新快照；CWA 歷史 API 在關鍵時段取得的是雷達回波，不是測站雨量歷線。

決策判斷矩陣

證據較弱

內湖本地雨下太大

內湖主要測站最高小時雨量約 20-24 mm/hr，未達 80-100 mm/hr 等級短延時暴雨。可作為負荷來源，但較不像單一主因。

證據較弱

基隆河主河道倒灌

南湖大橋 3.69 m，低於三級警戒 6.4 m；大直橋 1.67 m，低於三級警戒 3.3 m。主河道警戒不足以解釋全部淹水。

目前最合理

內水排除與流域瓶頸

文山、信義、南港山區長時間產流，與內湖本地降雨同時進入都市排水系統，較可能在箱涵、進水端、閘門或抽排能力形成瓶頸。

淹水形成鏈

山區連續降雨

文山、信義、南港山區多小時中高強度降雨。

土壤趨近飽和

新增降雨較難入滲，地表逕流比例升高。

逕流集中下洩

上游水量與內湖本地降雨在低地匯合。

箱涵或幹線滿流

若幹線流量超過容量，就會局部回堵。

抽排受限

抽水站滿載或基隆河高水位，會拉長積水時間。

最需要決策者追的資料

  • 1抽水站逐分鐘啟停紀錄、抽水量、內外水位歷線最高優先
  • 2成功路、港墘路、文德路幹線箱涵水位與流量定位瓶頸
  • 3基隆河排放口外水位與主河道水位曲線判斷回水
  • 4積淹水通報時間、最高積水深度與退水時間校正模型

O-A0001 / O-A0002 測站雨量

以淹水發生前的時間窗查詢，這兩組資料沒有可用歷史檔；目前無法用它們直接補出內湖、南港、文山、信義各測站的逐時雨量歷線。

決策限制

雷達回波不是雨量毫米

dBZ 可作為降雨系統佐證，但不能直接等同 mm/hr。工程判斷仍需取得 10 分鐘或逐時測站雨量、抽水站歷線與箱涵水位，才能校正淹水發生前的實際入流量。
